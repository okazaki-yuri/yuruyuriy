# スタイルガイド（デザイン / コーディング規約）

サイト全体で見た目と実装の一貫性を保つための指針。新しいページ・ツールを追加する際はこれに従う。

- 上位ドキュメント: [README](../README.md)
- 関連: [新しいツールの追加](./add-new-tool.md)

---

## 1. デザインコンセプト
- キーワードは「かわいい！！」。全体的に**クリーム・ベージュ系のやさしいトーン**でまとめる。
- カード型UI（角丸・淡い影・淡い背景色）を基本とする。
- ホバー時はやわらかい浮き上がり（`translateY`）や色反転で反応を返す。

## 3. タイポグラフィ
- フォント: **M PLUS Rounded 1c**（Google Fonts）。全ページで読み込む。
- 一部パネルで `Yu Gothic` / `Hiragino Kaku Gothic ProN` 系のフォールバックを併用。
- 見出し `h1` は上下に余白（`margin: 2rem 0`）。

## 4. レイアウト
- コンテンツ幅は `max-width` で制限（本文 `900px` 前後、パネル `600〜800px`、一覧 `1000px`）し、`margin: 0 auto` で中央寄せ。
- カード並びは `flex`（`flex-wrap`）または `grid`（`auto-fit` / `minmax`）でレスポンシブに。
- ブレークポイント:
  - `max-width: 768px` … ヘッダーのPCナビ→ハンバーガー切替（`header-footer.css`）。
  - `max-width: 600px` … ロゴ縮小・カード幅100%など（`css/style.css`）。

## 5. CSS 構成のルール
- **共通スタイルは `css/style.css`** に置く（トップ、ツール一覧、`.introduction-area` / `.howto-area` などツール共通パーツ）。
- **ヘッダー・フッターは `components/header-footer.css`**（全ページで読み込む）。
- **ページ/ツール固有のスタイルは各ディレクトリの `style.css`** に閉じる。共通で使えるものを個別CSSに重複させない。
- 読み込み順は各HTMLで「共通CSS → 専用CSS → ヘッダー・フッターCSS」を踏襲する。

### よく使う共通クラス（再利用する）
| クラス | 用途 |
| --- | --- |
| `.introduction-area` | ツールの概要パネル |
| `.howto-area` | 使い方ガイドパネル（`ul`/`li` で手順を列挙） |
| `.tool-card` / `.active-tool` / `.coming-soon` | ツール一覧のカード |
| `.link-card` | 外部リンクカード |

## 6. 命名規約
- CSSクラスは**英小文字＋ハイフン区切り（ケバブケース）**（例: `top-link-box`, `dice-card`）。
- ツールのディレクトリ名（slug）も同様にケバブケース（例: `web-dice-chan`）。
- IDは主にJSから参照する要素に付与（例: `#wordInput`, `#rollBtn`）。

## 7. マークアップ / アクセシビリティ
- 画像には `alt` を必ず設定する。
- 装飾外の画像には `width` / `height` 属性を付け、レイアウトシフト（CLS）を防ぐ。ファーストビュー外の画像は `loading="lazy"` を付与。
- 外部リンクは `target="_blank"` に加えて `rel="noopener"`（必要に応じて `noreferrer`）を付ける。
- 画像は表示サイズに合わせて最適化してから配置する（アイコンは128px程度、ロゴは480px程度）。

## 8. JavaScript
- フレームワークなしの Vanilla JS。ページ固有ロジックは各ツールの `script.js` に閉じる。
- 主要関数には用途を示すコメント（JSDoc風）を付ける（既存ツールに倣う）。
- 乱数は `Math.random()` を使用（暗号用途ではない）。
- 永続化は localStorage。キー名は一意にし、個人情報は保存しない。
