# スタイルガイド（デザイン / コーディング規約）

サイト全体で見た目と実装の一貫性を保つための指針。新しいページ・ツールを追加する際はこれに従う。

- 上位ドキュメント: [README](../README.md)
- 関連: [新しいツールの追加](./add-new-tool.md)

---

## 1. デザインコンセプト
- キーワードは「かわいい！！」。全体は**グレー×クリームの柔らかいトーン**をベースに、**くすみカラー（低彩度・ダスティ）**のアクセントで落ち着いたやさしさを出す。
- カード型UI（角丸・淡い影・淡い背景色）を基本とする。
- ホバー時はやわらかい浮き上がり（`translateY`）や色反転で反応を返す。
- アクセントは**くすみローズ（ダスティローズ）を主役**とし、鮮やかな色（かつてのオレンジ・青・黒など）は使わない。色を足したいときは低彩度・中〜暗トーンに寄せる。

## 2. カラーパレット（デザイントークン）

### 2.1 方針
- 色・角丸・影は **CSS カスタムプロパティ（デザイントークン）で一元管理**する。定義箇所は **`apps/web/app/styles/style.css` の `:root`**（全ページで最初に読み込まれるため全域で参照可能）。
- 各ページ／コンポーネントの CSS は**生の 16進数（hex）を直書きせず**、必ず `var(--token)` を参照する。**色を変えるときは `:root` の値だけを書き換える**（一箇所で全ページに反映される）。
- 例外は `header-footer.css` のフッターのグラデーション端点のみ（透明フェードのため rgba 直値。値は `--surface-alt` と一致させる）。

### 2.2 トークン一覧
| トークン | 値 | 用途 |
| --- | --- | --- |
| `--bg` | `#f5f2ec` | ページ背景（温かみのあるクリームグレー） |
| `--surface` | `#fdfbf7` | カード面（ほぼ白のクリーム） |
| `--surface-alt` | `#f7f4ee` | 補助面・ヘッダー・概要パネル |
| `--border` | `#e5ded3` | やわらかい罫線 |
| `--text` | `#4a4642` | 本文（くすんだ墨色） |
| `--text-muted` | `#6f6a62` | 補助テキスト |
| `--text-faint` | `#8a837b` | 註釈など**大きめ文字専用**（小文字には使わない） |
| `--primary` | `#7f4f56` | 主アクセント（くすみローズ）。ボタン・タブ・枠線など |
| `--primary-hover` | `#6d4046` | 主アクセントのホバー |
| `--on-primary` | `#ffffff` | 主アクセント上の文字色 |
| `--accent` | `#7f8c7a` | 副アクセント（スモーキーセージ）。副次的アクションに使用 |
| `--accent-hover` | `#6b7767` | 副アクセントのホバー |

### 2.3 色の使い分け
- **主アクション**（実行・回す・振る・使ってみる・ナビ・タブ選択）＝ `--primary`。
- **副次アクション**（リセット・履歴クリアなど「元に戻す／消す」系）＝ `--accent`。役割を色で区別する。
- ボタンのホバーは**白地（`--surface`）に文字を主アクセント色へ反転**、が基本パターン。
- カード・パネルの背景は `--surface` / `--surface-alt`、枠は `--border`、影は `--shadow-card`（ホバーで `--shadow-hover`）。

### 2.4 かたちのトークン
| トークン | 値 | 用途 |
| --- | --- | --- |
| `--radius` | `10px` | カード・パネルの角丸（標準） |
| `--radius-sm` | `6px` | ボタン・入力欄などの角丸 |
| `--radius-pill` | `999px` | タグなどピル形 |
| `--shadow-card` | `0 4px 10px rgba(74,70,66,0.06)` | カードの通常影 |
| `--shadow-hover` | `0 6px 16px rgba(74,70,66,0.12)` | ホバー時の影 |

### 2.5 アクセシビリティ（コントラスト）
- テキスト・UI色は原則 **WCAG 2.1 AA（本文 4.5:1／大きな文字・UI 3:1）以上**を満たすよう選定している。
  - `--text` on `--bg`：約 **8.4:1**（AAA）
  - `--text-muted` on `--bg`：約 **4.8:1**（AA）
  - `--primary` に `--on-primary`（白文字）：約 **6.7:1**（AAA 目前）
- **`--text-faint`（`#8a837b`）は小さい本文に使わない**（コントラスト不足）。プレースホルダや大きめの註釈に限定。
- **新しい色を追加・変更したら、白文字／背景とのコントラスト比を必ず確認**し、上記基準を割らないこと。

## 3. タイポグラフィ

### 3.1 フォント
- フォント: **M PLUS Rounded 1c**（Google Fonts）。丸ゴシックの柔らかさがコンセプトの要。
- 読み込みは `apps/web/app/layout.tsx` で `wght@400;500;700` を**明示指定**する。ウェイトを指定しないと 400 のみ配信され、見出し・ボタンが**合成ボールド（faux bold）**になって輪郭が潰れるため必ず指定する。
- `font-family` は**トークン `--font-sans` を参照**する（個別CSSで直書きしない）。読み込み前（`display=swap`）や失敗時に備え、日本語の丸ゴ／ゴシック系システムフォントをフォールバックに含める:
  ```
  --font-sans: 'M PLUS Rounded 1c', 'Hiragino Maru Gothic ProN',
    'Hiragino Kaku Gothic ProN', 'Yu Gothic', Meiryo, sans-serif;
  ```
- **個別パネルで別フォントに上書きしない**（以前あった `Yu Gothic` への上書きは廃止済み）。

### 3.2 ウェイト
`--fw-normal: 400` / `--fw-medium: 500` / `--fw-bold: 700` のみを使う。`font-weight` はキーワード（`bold` 等）でなく**トークンで指定**し、読み込み済みの3ウェイトの範囲に収める。

### 3.3 タイプスケール（トークン）
サイズは **`rem` に統一**し、以下のスケールから選ぶ（`em`／`px` の直書きはしない）。

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--fs-2xl` | `2rem` | ページ見出し（h1 相当）・ルーレット結果 |
| `--fs-xl` | `1.8rem` | セクション大見出し |
| `--fs-lg` | `1.3rem` | カード見出し |
| `--fs-md` | `1.2rem` | パネル見出し・ツール名 |
| `--fs-base` | `1rem` | 本文 |
| `--fs-sm` | `0.9rem` | 補助・説明文 |
| `--fs-xs` | `0.85rem` | 註釈・小ラベル |
| `--fs-2xs` | `0.75rem` | コピーライトなど極小 |

### 3.4 行間
- `body` に基準 `line-height: var(--lh-base)`（`1.7`）を設定。日本語本文は行間広めが読みやすい。
- 見出し（`h1`〜`h3`）は `--lh-heading`（`1.3`）で詰める。
- 見出し `h1` は上下に余白（`margin: 2rem 0`）。

## 4. レイアウト

### 4.1 コンテナ戦略
- **`body` には `max-width` を掛けない**（全幅）。掛けるとヘッダー／フッターの全幅背景や各ページの幅指定が頭打ちになるため。
- **ヘッダー／フッターは背景を画面全幅**にし、**中身だけ `--content-wide` のインナーコンテナ**（`.header-inner` など）で中央寄せする。
- 各 `main`（ページ）は自前の `max-width` トークン＋`margin: 0 auto` で中央寄せする。
- 横幅は**トークンから選ぶ**（生の px を直書きしない）。左右の最小余白は `--gutter`。

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--content-wide` | `1000px` | ヘッダー・フッター中身・ツール一覧 |
| `--content` | `960px` | 標準の本文カラム（`main`） |
| `--reading` | `800px` | 読み物・ヒーロー列（TOP・お問い合わせ・規約） |
| `--panel` | `600px` | パネル・入力エリア（`.introduction-area` / `.howto-area` / ツールの入力欄など） |
| `--gutter` | `1rem` | コンテナ左右の最小余白 |

### 4.2 余白スケール
ページ構造の縦リズムは `--space-*` から選ぶ（`px`／`em` の直書きはしない）。

| トークン | 値 |
| --- | --- |
| `--space-xs` | `0.5rem` |
| `--space-sm` | `1rem` |
| `--space-md` | `1.5rem` |
| `--space-lg` | `2rem` |
| `--space-xl` | `3rem` |

### 4.3 並べ方・レスポンシブ
- カード並びは `flex`（`flex-wrap`）または `grid`（`auto-fit` / `minmax`）＋ `gap` でレスポンシブに。兄弟間隔は個別マージンでなく `gap` を使う。
- ブレークポイント:
  - `max-width: 768px` … ヘッダーのPCナビ→ハンバーガー切替（`header-footer.css`）。
  - `max-width: 600px` … ロゴ縮小・カード幅100%など（`styles/style.css` および各ツールCSS）。

## 5. CSS 構成のルール
- **共通スタイルは `apps/web/app/styles/style.css`** に置く（トップ、ツール一覧、`.introduction-area` / `.howto-area` などツール共通パーツ）。
- **ヘッダー・フッターは `apps/web/app/styles/header-footer.css`**（ルートレイアウトで全ページに読み込む）。
- **ページ/ツール固有のスタイルは各ページディレクトリのCSS**（例: `apps/web/app/tools/web-dice-chan/dice.css`）に閉じ、各 `page.tsx` から import する。共通で使えるものを個別CSSに重複させない。

### よく使う共通クラス（再利用する）
| クラス | 用途 |
| --- | --- |
| `.introduction-area` | ツールの概要パネル |
| `.howto-area` | 使い方ガイドパネル（`ul`/`li` で手順を列挙） |
| `.tool-card` / `.active-tool` / `.coming-soon` | ツール一覧のカード |
| `.link-card` | 外部リンクカード |

## 6. 命名規約
- CSSクラスは**英小文字＋ハイフン区切り（ケバブケース）**（例: `top-link-box`, `dice-card`）。
- ツールのディレクトリ名（slug）も同様にケバブケース（例: `web-dice-chan`）。
- IDは主にJSから参照する要素に付与（例: `#wordInput`, `#rollBtn`）。

## 7. マークアップ / アクセシビリティ
- 画像には `alt` を必ず設定する。
- 装飾外の画像には `width` / `height` 属性を付け、レイアウトシフト（CLS）を防ぐ。ファーストビュー外の画像は `loading="lazy"` を付与。
- 外部リンクは `target="_blank"` に加えて `rel="noopener"`（必要に応じて `noreferrer`）を付ける。
- 画像は表示サイズに合わせて最適化してから配置する（アイコンは128px程度、ロゴは480px程度）。

## 8. JavaScript / TypeScript
- Next.js（App Router）+ TypeScript。ページ固有のインタラクションは各ツールのクライアントコンポーネント（例: `WordRoulette.tsx` / `WebDice.tsx`）に閉じる。
- 抽選・計算・検証などの**純粋ロジックは `@yuruyuriy/core`（`packages/core/src/`）** に置き、UIから import する（[add-new-tool.md](./add-new-tool.md) 参照）。
- 主要関数には用途を示すコメント（JSDoc風）を付ける（既存ツールに倣う）。
- 乱数は `Math.random()` を使用（暗号用途ではない）。
- 永続化は localStorage。キー名は一意にし、個人情報は保存しない。
